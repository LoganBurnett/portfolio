Online Storefront

This is an ASP.NET site for an online storefront that incorporates user identity allowing for different functionality based on admin permissions. Customers can add items to their cart, increase/decrease each items quantity, and checkout their order. Admins can update and add new products, categories, and manufacturers as well as manage users and user roles. The site also has full Stripe functionality to process payments and uses an email sender to confirm customer's orders.

California Speed Game

This is an app for playing the card game California Speed, which incorporates drag-and-drop functionality for moving the cards into piles. The app also uses sockets to let two users on the same network play the game against each other. Once the game finishes, the players are each asked for their names, then that game's information is stored in a Mongo database and displayed for the users to see.

    My Contributions:
  • The drag-and-drop functionality and display for the cards and piles
  • All of the sockets functionality
  • The game history page
  • Some of the game logic, including the card shuffling algorithm

One of the technical challenges we overcame during this project was differentiating between the players, which we solved by using each player's socket id in a key value pair with their player number.

Banking App

This is a relatively simplified concept for a banking app. The user can sign up and log into an account, which provides them with a checking, savings, and credit account. The user can then choose to deposit, withdraw, and/or transfer money between their accounts, which will then show up on their transaction history page. The app uses sha256 hashing and salting on each user's password and stores them in a MongoDB database. It also uses sessions to track the user across pages and ensure that their secure information is only accessible when they are properly logged in.

    My Contributions:
  • The sign up and login page
  • The accounts page
  • The hashing, salting, storing, and validating of passwords
  • All of the session functionality
  • The backend routes for handling deposits, withdraws, and transfers
